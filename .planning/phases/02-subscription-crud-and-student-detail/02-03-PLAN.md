  <action>
    Create components/SubscriptionSection.tsx as a "use client" component.

    Props: { studentId: string; initialSubscriptions: PlatformSubscriptionWithCharges[] }

    State:
    - subscriptions: PlatformSubscriptionWithCharges[] (initialised from props)
    - showAddForm: boolean (default false)
    - editingId: string | null (which subscription is being edited inline; null = none)
    - saving: boolean (submit in progress)
    - error: string

    Add-form fields (controlled inputs):
    - amountCHF: string (number input, min=0, step=0.5)
    - durationMonths: "1" | "6" (select)
    - billingMethod: "invoice" | "direct" (select)
    - startMonth: number (select using monthOptions from ui-format.ts)
    - startYear: number (input, default current year)
    All default to getCurrentMonthYear() for startMonth/startYear on mount.

    Edit-form fields (per subscription, shown inline when editingId === sub.id):
    - amountCHF: string
    - billingMethod: "invoice" | "direct"

    Section header:
    - "Mathetogo Abonnement" heading (text-sm font-medium text-gray-700)
    - "Hinzufügen" button — disabled and hidden when an active subscription already exists (check subscriptions.some(s => s.active))
    - Show "Bereits ein aktives Abonnement vorhanden" note instead when active exists

    Per-subscription display:
    - Status badge: active=true → "Aktiv" bg-green-100 text-green-700; active=false → "Inaktiv" bg-gray-100 text-gray-500
    - Monthly amount: formatCHF(sub.amountCHF) + "/Monat"
    - Billing method label: billingMethod === "invoice" → "Rechnung" ; "direct" → "Überweisung"
    - Months remaining: monthsRemaining(sub) + " Monate verbleibend" (or "Abgelaufen" if 0)
    - Charge rows: buildChargeRows(sub, sub.charges) → render as a small table or list:
      - Columns: Month/Year label (use monthOptions to get German name), Amount CHF, Status badge
      - Status labels: paid → "Bezahlt" (green), unpaid → "Ausstehend" (red), scheduled → "Geplant" (gray)
    - When all rows are scheduled (no actual charges yet): show note below the list: "Abrechnungen werden bei Rechnungserstellung erstellt."
    - Actions: "Bearbeiten" button (only when sub.active); "Deaktivieren" button (text-red-500, only when sub.active)

    Inline edit form (shown when editingId === sub.id):
    - amountCHF input + billingMethod select
    - "Speichern" button → PATCH /api/subscriptions/[sub.id] with { amountCHF, billingMethod }; on success update subscriptions state, set editingId to null
    - "Abbrechen" button → set editingId to null

    Deactivate handler:
    - window.confirm("Abonnement wirklich deaktivieren? Dies stoppt zukünftige Abrechnungen.") → if confirmed: PATCH /api/subscriptions/[sub.id] with { active: false }; on success update subscriptions state

    Add-subscription submit handler:
    - POST /api/subscriptions with form values
    - On 409: show "Student hat bereits ein aktives Abonnement" in error state
    - On success: prepend new subscription to subscriptions state, reset form, set showAddForm to false

    All fetch calls:
    - Use method: "POST"/"PATCH", headers: { "Content-Type": "application/json" }, body: JSON.stringify(...)
    - Set saving=true before fetch, saving=false in finally
    - Set error on non-ok response

    Component structure (outer shell):
    ```tsx
    <section className="rounded-2xl border border-blue-100 bg-white shadow-sm p-4">
      <div className="mb-3 flex items-center justify-between">
        <h3 className="text-sm font-medium text-gray-700">Mathetogo Abonnement</h3>
        {!hasActive && (
          <button onClick={() => setShowAddForm(true)} className="text-sm text-[#4A7FC1] hover:underline">
            Hinzufügen
          </button>
        )}
      </div>
      {/* add form, subscription list */}
    </section>
    ```

    Keep the component focused. Do not add animations or modals — use inline show/hide via state booleans only (per RESEARCH.md open question resolution).
